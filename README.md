# slm-humor-distill

Distill a Qwen2.5-14B teacher into a Qwen2.5-1.5B student specialized for short Japanese humor one-liners, using synthetic data + MLX LoRA on Apple Silicon.

This is an educational / experimental pipeline. It does **not** produce a production humor model, and it does **not** reproduce Sakana AI's TAID (logit-level distillation). The approach here is sequence-level distillation: generate teacher answers, then supervise the student with LoRA.

## What it does

- Generate teacher one-liners with Qwen2.5-14B (4-bit)
- Fine-tune Qwen2.5-1.5B (4-bit) with MLX LoRA
- Compare before / after on the same prompts
- Benchmark local inference throughput

## Models

| Role | Model | MLX Community |
|---|---|---|
| Teacher | Qwen2.5-14B-Instruct | `mlx-community/Qwen2.5-14B-Instruct-4bit` |
| Student | Qwen2.5-1.5B-Instruct | `mlx-community/Qwen2.5-1.5B-Instruct-4bit` |

## Requirements

- macOS + Apple Silicon (M1 or later)
- Python 3.12 (`uv` recommended)
- Roughly 16GB+ free unified memory for teacher generation / LoRA

## Setup

```bash
git clone https://github.com/masanori0209/slm-humor-distill.git
cd slm-humor-distill

uv venv --python 3.12 .venv
source .venv/bin/activate
uv pip install -r requirements.txt
```

## Run everything

```bash
bash scripts/run-all.sh
```

The first run downloads the teacher and student weights from Hugging Face.

## Step by step

```bash
# 1. Generate teacher dataset
python scripts/generate_teacher_data.py

# 2. Split train / valid
python scripts/prep_train_data.py

# 3. LoRA fine-tuning
bash scripts/train_lora.sh

# 4. Fuse adapter into base model
bash scripts/fuse_model.sh

# 5. Before / after comparison
python scripts/eval_before_after.py --fused-model fused_model/humor-1.5b-fused
python scripts/bench_mlx.py --model fused_model/humor-1.5b-fused
```

MLX uses Metal on Apple Silicon. For strict CPU-only inference, convert / download a GGUF and run `llama.cpp` with `-ngl 0`.
